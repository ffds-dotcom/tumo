Day 3 / GitHub Setup

GitHub Setup

Connect your project to GitHub so your work is saved online. Follow these steps once per computer, then push in seconds.

Steps

0. Create Account 1. Create Token 2. Init / Clone 3. Config Name & Email 4. Set Remote URL 5. Push Code 6. Each New Session
0
Create a GitHub account

If you don't have a GitHub account yet, create one now. Use your TUMO email address so your instructor can find you.

Go to github.com → click Sign up → enter your TUMO email
Tip: Choose a username you're happy to keep — it will appear on all your public projects.

Already have an account? Skip to Step 1.

1
Create a Personal Access Token on GitHub

Navigate to your token settings by following this path:

GitHub → Settings → Developer settings → Personal access tokens → Tokens (classic) → Generate new token (classic)

Under Select scopes, check only:

✔
repo
Full control of private repositories (includes push, pull, clone)

Set expiration to 30 days. Then click Generate token and copy it immediately — GitHub only shows it once.

Warning: Do not close the page before copying the token. If you lose it, you will need to generate a new one.
2
Clone or initialize your repository

If you are starting a new repository from scratch:

terminal — new repo
git init git add . git commit -m "first commit" git branch -M main

If you are cloning an existing repository, skip to Step 3.

3
Configure Git with your name and email

Run these two commands inside your repository folder:

terminal
git config --local user.name "Your Name" git config --local user.email "your@email.com"

Using --local keeps this setting scoped to this repo only and won't affect other accounts on the computer.

4
Set the remote URL with your token embedded

This is the key step. Fill in the fields above, then click this block to preview it in the terminal:

terminal
git remote add origin https://USERNAME:YOUR_TOKEN@github.com/USERNAME/REPO_NAME.git

If a remote already exists (e.g. you cloned the repo), use set-url instead:

terminal — update existing remote
git remote set-url origin https://USERNAME:YOUR_TOKEN@github.com/USERNAME/REPO_NAME.git
Note: Your token is visible in plain text inside .git/config. Do not share your screen or push .git/config to GitHub. Since the lab computers wipe on inactivity, this is generally acceptable.
5
Push your code

You should now be able to push without being prompted for credentials:

terminal
git push -u origin main

For all future pushes in the same session, just run:

terminal
git add . git commit -m "your message" git push
6
Each new session — run this one command

Because the computers wipe on inactivity, you need to re-set the remote URL each time. Save this command somewhere accessible — a notes app, your email, etc.

run once per session
git remote set-url origin https://USERNAME:YOUR_TOKEN@github.com/USERNAME/REPO_NAME.git
No SSH keys needed No password prompts Re-run each session
