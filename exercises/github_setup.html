GitHub Setup

Push Your Code to GitHub

GitHub is a website that stores your code online so it's safe even when the computer is wiped. Follow these steps to push your my-project to your own GitHub account.

No SSH needed

We use HTTPS + a Personal Access Token — no SSH key setup required. The token is like a one-time password.

Your own repo

Each student pushes to their own GitHub account. The URL will be github.com/YOUR-NAME/my-project.

Save your token

You see the Personal Access Token only once. Write it down or save it in your phone notes before closing the page.

Steps

1. Sign in to GitHub 2. Create a repository 3. Copy your URL 4. Connect your folder 5. Get a token 6. Push your code 7. Verify it worked ⚠ Fix 403 error

Step 1 — Sign in to GitHub

1

Open github.com in your browser

Go to github.com and sign in with your account. If you do not have an account, click Sign up and create one now.

Tip Use a personal email you can always access — not a school email that may expire.

Step 2 — Create a new repository

2

Click "Create repository" on your dashboard

After signing in you will see your GitHub dashboard. Click the green Create repository button.

Fill in the form like this:

Required fields
Owner: your personal account
Repository name: my-project
Visibility: Public
README / .gitignore / license: leave all OFF

Then click the green Create repository button at the bottom of the form.

Step 3 — Copy your HTTPS URL

3

Find and copy the repository URL

After creating the repo GitHub shows a "Quick setup" page. Make sure HTTPS is selected (not SSH), then copy the URL. It looks like this:

https://github.com/YOUR-USERNAME/my-project.git
Where to find it It's on the empty repo page right after you create it — look for the HTTPS button and the copy icon next to the URL.

Step 4 — Connect your local folder to GitHub

4

Run these two commands in your my-project folder

Replace YOUR-USERNAME with your actual GitHub username:

git remote add origin https://github.com/YOUR-USERNAME/my-project.git
git branch -M main
What this does git remote add origin tells your local repo where to send code. git branch -M main renames your branch to "main" so it matches GitHub.

Step 5 — Create a Personal Access Token

GitHub no longer accepts your account password — you need a token instead.

5

Generate a token on GitHub

Follow this path on GitHub:

Click your profile picture (top-right) Settings Developer settings (bottom of the left sidebar) Personal access tokens → Tokens (classic) Generate new token → Generate new token (classic)

Fill in the form:

Token settings
Note: my-project token
Expiration: 90 days
Scope: check the box next to repo

Click Generate token. You will see a green token starting with ghp_.

⚠ Save your token now GitHub shows the token only once. Copy it and save it in your phone notes or a text file before closing this page. You will paste it as your password when you push.

Step 6 — Push your code

6

Run the push command

In your terminal, inside the my-project folder, run:

git push -u origin main

Git will ask for your credentials:

What to type when prompted
Username: your GitHub username (e.g. CT1234)
Password: paste your token (starts with ghp_) — NOT your real password
The token won't appear as you type That's normal. Paste it and press Enter even if you see nothing.

Step 7 — Verify it worked

7

Check GitHub in the browser

Open your browser and go to:

https://github.com/YOUR-USERNAME/my-project

You should see your files — chess.js, index.html, style.css — listed on the page. Your code is now safely backed up online.

Next time you make changes Run git add . → git commit -m "message" → git push. No need to set up the remote again.

Quick Checklist

⚠ Troubleshooting: 403 Permission Error

!

You see this error when you push

remote: Permission to someone-else/their-project.git denied to YOUR-USERNAME. fatal: unable to access 'https://github.com/someone-else/their-project.git/': The requested URL returned error: 403
What this means Your origin remote is pointing at someone else's repo. You don't have write permission there. The fix is to point origin at your own repo.

Fix — update your remote URL

Run this, replacing YOUR-USERNAME with your actual GitHub username:

git remote set-url origin https://github.com/YOUR-USERNAME/my-project.git
git push -u origin main
!

Git doesn't ask for username / password at all

Old credentials may be cached. Clear them and try again:

git config --global credential.helper ""
git push -u origin main

Now git will prompt for your username and token.

!

Password rejected even with the token

Check two things:

  • The token starts with ghp_ — if you pasted your real GitHub password it won't work.
  • The token has the repo scope checked. If you forgot, generate a new one.
  • The token hasn't expired — if it has, generate a new one.
